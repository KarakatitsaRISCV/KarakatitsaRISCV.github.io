Инструкция по заполнению файлов .pintable

Список разделов:

'Packages:' Опциональный, однострочный. Список корпусов (имена в формате KiCad) и сокрощенных псевдонимов для них. Разные корпуса разделяются точкой с запятой. Например,
Packages: TSSOP20[TSSOP-20_4.4x6.5mm_P0.65mm]; QFN20X3[QFN-20-1EP_3x3mm_P0.4mm_EP1.65x1.65mm]

'MCU:' Обязательный, однострочный. Список названий микросхем и их корпуса (либо сокрощенное нозвание, либо полное). Например:
MCU: CH32V203F6P6[TSSOP20]; CH32V203F8P6[TSSOP20]; CH32V203F8U6[QFN20X3]

'Periph:' Опциональный, однострочный. Список периферии (возможно с сокрощенным вариантом), для управления отображением. В квадратных скобках можно указать сокрощенное имя. Далее в таблице одно из них, плюс точка, плюс строковое описание, будут служить идентификатором, к какой периферии вывод относится. Например:
Periph: GPIO[P]; TIM; Analog[A]; UART; SPI
Здесь у TIM, UART и SPI есть только полные имена. В таблице они будут выглядеть как TIM.1.2, SPI.1_MOSI, UART.TX и т.д. У GPIO и Analog же есть сокрощенные имена. К ним можно обращаться как A.1, P.PB7 и т.д. Если какая-то периферия доступна в виде ремапа, в таблице ее имя указывается в квадратных скобках, после которых идет номе ремапа. Например, UART.[2_RX]3.

'Comment' Опциональный, однострочный. Именованная сноска. После слова 'Comment' без пробела идет номер или имя сноски. В таблице на нее можно будет сослаться через COMM[n], где n - имя сноски. Например,
CommentT1 Some text
Здесь имя сноски - 'T1'

# Опциональный, однострочный. Любая строка (кроме области Content), начинающаяся с решетки, считается комментарием, и программой игнорируется.

'Content:' Обязательный, многострочный. Таблица выводов и их функций. Чем-то похожа на таблицу markdown. Столбцы отделяются вертикальными палками ('|').
Первая строчка - заголовок. Первый элемент - 'Name', имя вывода. Далее идет один или несколько элементов для микросхем или их корпусов. Имя микросхемы указывается просто так, имя корпуса - в квадратных скобках. Если несколько микросхем нужно разместить в одном столбце, они разделяются точкой с запятой. Последний столбец - 'Periph', список периферии, относящейся к данному выводу.
Вторая строчка - разделитель (не анализируется)
Остальные строчки - данные.
Первый столбец - имя вывода (короткая строка).
Далее один или несколько столбцов - номера выводов для каждой из указанных микросхем. Если одному и тому же имени соответствует несколько выводов, их можно перечислять через точку с запятой. Если в данном корпусе этого вывода нет, можно поставить минус ('-').
Последний столбец - список периферии, разделенной точкой с запятой. Если периферия доступна через ремап, см. выше, имя указывается в квадратных скобках, после которых может идти номер ремапа. Если периферия не относится ни к какой из указанных ранее, она пишется как есть, без лишних точек, и попадает впоследствии в столбец 'Other'. Специальный "тип периферии" - SRV, служебные выводы (земли, питания и т.д.). Могут обозначаться как SRV.F.pin или SRV.O.pin. Буква F (fixed) обозначает, что вывод pin работает только как служебный вывод. Буква O (optional) - что может быть переназначен на какую-нибудь другую функцию.
Опциональный столбец, не входящий в таблицу и не оканчивается вертикальной палкой - комментарий или сноска. Если необходимо сослаться на длинный текст после таблицы, пишем COMM[1], COMM[T1] или что-то подобное. В html-файле это будет выглядеть как ссылка в поле имени вывода, ведущая в область комментариев. Таких сносок может быть несколько, разделенных точкой с запятой. Текст, не являющийся COMM[n], вставляется в html-файл в виде всплывающей подсказки.
Концом таблицы считается первая строка, не начинающаяся с вертикальной палки.
Пример:
Content:
| Name    | [QFN20X3] | [LQFP32] | [LQFP48];[QFN48X7] | Periph |
|---------|------------------------------------------|--------|
|   GND   |    21     |    -     |         49        | SRV.F.GND  |
|   Vbat  |    -      |    -     |          1        | SRV.F.VCC | Вывод питания
|   PC13  |    -      |    -     |          2        | P.PC13; TAMPER_RTC |
|   PC14  |    -      |    -     |          3        | P.PC14; SRV.O.OSC32_IN | 
|   PC15  |    -      |    -     |          4        | P.PC15; SRV.O.OSC32_OUT |
|   PB8   |    -      |    31    |          45       | P.PB8; 5.+; TIM.4.3; I2C.[1_SCL]; CAN.[1_RX] | COMM[T1]
|   PB9   |    -      |     -    |          46       | P.PB9; 5.+; TIM.4.4; I2C.[1_SDA]; CAN.[1_TX] |
